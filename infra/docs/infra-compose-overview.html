The Problem

1.
Every project starts from scratch
Copy-paste docker-compose from old projects or Slack. Different structure every time. No consistency.
2.
Switching test data takes forever
Need different test data? Wipe everything, re-seed, wait minutes. Hope you didn't need the old data.
3.
Port conflicts break everything
Two projects using port 5432? Cryptic startup errors. Nobody knows which container to stop.
2 / 9

The Solution

A shared library of database templates.
Pick the services you need. 3 lines of config. Done.

3 / 9

Profile Switching

Think of profiles like save slots in a video game. Each slot has different data. Loading is instant.

small
1 org, 2 users
Fast iteration & debugging
basic
5 orgs, 50 users
Realistic testing
large
100 orgs, 10K users
Load & performance testing
$ npx infra-compose switch --profile small ✓ Switched
~30s
First load (seeds data)
~3s
After that (instant swap)

Your old data is never lost. Just load a different slot.

How It Works

5 / 9

3 Steps to Start

Step 1

Install

pnpm add -D @saga-ed/infra-compose
Step 2

Extend

# docker-compose.yml
services:
  postgres:
    extends:
      file: node_modules/@saga-ed/
        infra-compose/services/
        postgres/compose.yml
      service: postgres
Step 3

Run

npx infra-compose up \
  --profile small

# Databases are running
# with test data loaded

Command Cheat Sheet

Command What it does
npx infra-compose up --profile small Start databases with the "small" dataset
npx infra-compose switch --profile basic Switch to a different dataset (instant after first load)
npx infra-compose down Stop everything (your data is preserved)
npx infra-compose reset --profile small Wipe one dataset and reload it fresh
npx infra-compose status Show what's currently running
